Issue:

In Word Puzzle mode, if a user taps a word in the Answer Area very quickly multiple times, the same word can appear multiple times in the Word Bank.

Result:

The word becomes duplicated and the puzzle state becomes invalid.

Task:

Review:

* handleBankWordClick
* handleAnswerWordClick

Ensure a word can only exist in ONE location at a time:

* Word Bank
    OR
* Answer Area

Never both.

Add protection against rapid repeated clicks and race conditions caused by React state updates.

Verify:

* Slow tapping
* Fast tapping
* Repeated tapping
* Mobile tapping

When these are placed correctly in Arrange Ayahs mode, the puzzle still reports:

Review Arrange Ayahs answer-checking logic.

Instead compare using unique identifiers such as:

The system must correctly distinguish between repeated ayahs.
